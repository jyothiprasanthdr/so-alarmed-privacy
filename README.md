# So Alarmed — Privacy Policy

The official privacy policy for **So Alarmed**, a privacy-first alarm app for iPhone
developed by **Nandha Kumar Ilangovan**.

## Live policy

Once GitHub Pages is enabled for this repository, the policy is served at:

**https://jyothiprasanthdr.github.io/so-alarmed-privacy/**

Use that URL as the **Privacy Policy URL** in App Store Connect.

## Files

| File | Purpose |
|---|---|
| [`index.html`](index.html) | The hosted, styled policy served by GitHub Pages. This is the canonical published page. |
| [`privacy-policy.md`](privacy-policy.md) | A plain-text Markdown mirror of the same policy. |

> Keep `index.html` and `privacy-policy.md` in sync when you make changes.

## Updating the policy

1. Edit `index.html` (and mirror the change in `privacy-policy.md`).
2. Update the **"Last updated"** date in both files.
3. Commit and push to `main`. GitHub Pages redeploys automatically.

## Notes

- The policy reflects the app's actual data practices as of June 28, 2026: optional
  Apple/Google sign-in via Firebase Auth, optional cloud backup of alarms + streak via
  Cloud Firestore, Apple StoreKit subscriptions, and on-device-only motion sensing for
  the Shake/Walk missions. No analytics, ads, crash reporting, or third-party tracking.
- If you add or remove any data collection (e.g. the planned "calendar wake" feature),
  update this policy **and** your App Store privacy disclosures before shipping.
